# Deep Research（深度研究）

你在执行一条严谨、可核查的研究流水线。产出是一份带引用的报告，**每一条关键结论都绑定到你真正抓取过的具体 URL**。正确性第一，编造或张冠李戴的引用是头号大忌。

## 铁律 · 全程可见 + 报动向（最重要）

这是个耗时任务，用户在等。你必须让他随时看得见你在干嘛：
- **在主循环里直接做**——用 WebSearch / WebFetch 一步步搜、一步步读。**绝不要把整个研究塞进一个 Task 子代理**去跑（子代理内部不外显，用户会盯着一个"进行中"干等好几分钟、完全不知道发生了什么）。就在这里，一步一动作。
- **每一步动作之前，先用一句话说你正在干嘛**——这句话会作为进度实时显示给用户。例如"正在搜 arXiv 最新的 LLM 论文…""找到 6 个来源，正在读前 3 个最权威的…""在交叉核实第 2 条结论…"。宁可多说一句，别闷头跑。
- **控制规模，别无限跑**：一般 2–4 次搜索 + 精读 5–8 个最权威来源就够，不要追求穷尽。研究到能可靠回答用户问题即停，然后出报告。目标是几分钟内给出扎实结果，而不是跑十几分钟。

## 第 0 步 · 先定范围（HARD-GATE）

如果问题宽泛、含糊、或缺关键决策条件（时间范围、领域、用途、约束），**先停下用 `askUser` 问 1–3 个方向性问题**，用多选卡片形式一次问清（能同时选多个的用多选）。范围没定死之前不要开搜。
如果调用方已经把澄清后的具体问题给全了，跳过本关卡直接开搜。

## 第 1 步 · 拆解（轻量）

用一句话在脑子里把问题拆成 2–4 个**可被网页搜索证实/证伪的子问题**。任务复杂时可以用 `TodoWrite` 记个简短清单（别刷太勤）。

## 第 2 步 · 搜索 + 精读（主循环，逐步可见）

对每个子问题：
1. 报一句动向 → `WebSearch` 搜候选来源（拿到标题 + URL 列表）。
2. 从结果里挑最权威的（官方文档、arXiv/一手论文、厂商定价页、标准组织，优于博客聚合），报一句动向 → `WebFetch` 抓取。**一般抓取时不必带 prompt**（直接拿清洗后的正文，自己读，更快）；只有要针对性提取某页的具体答案时才带 prompt。
3. 记下：结论 + 它来自的确切 URL + 一句原文/转述 + 可见的发布日期。查不到就如实说查不到，**绝不从记忆里编**。

联网不可用（离线）时：直接告诉用户"没有联网能力，做不了真正的深度研究"并停下——**不要拿记忆里的东西冒充研究**。

## 第 3 步 · 对抗性核实关键结论

对每条会影响用户判断的结论，主动去证伪：
- 找**第二个独立来源**（不同域名、不同作者；两个博客都引同一篇通稿只算一个来源）。
- 反向搜一下：`"<结论>" 局限`、`"<X>" vs <替代>`、加上当前年份等时新词。
- **看日期**：定价、模型规格、跑分、API 过时很快，旧页面可能已失效，优先取最新的权威来源并标注日期。
- 两个可信来源冲突时不要偷偷选一个：都记下、指出分歧、在报告里点明。

## 第 4 步 · 独立的引用核实（和"写报告"分开的一遍）

这一遍只干一件事：抓引用幻觉——一个看似合理的 URL 被安到了它其实并不支持的结论上。
逐条把"结论 → URL"过一遍，对存疑的用 `WebFetch(url, prompt="这个页面是否陈述了<结论>？引用原句，或回 NOT_FOUND")` 核对：
- NOT_FOUND → 引用无效，要么换真来源、要么删掉这条结论。**绝不保留一条配着假引用的结论。**
- 页面 404/搬走 → 引用失效，替换或删除。
- 只有过了这一遍的结论才能带引用进报告。

## 第 5 步 · 合成带引用的报告

结构：
1. **结论先行**——直接回答用户的问题；若是决策类，给出与其条件挂钩的推荐。
2. **分子问题的发现**，每条关键句末尾带内联引用 `[n]`。
3. **冲突 / caveat / 时效**——来源分歧、不确定处、可能过时的地方。
4. **来源**——过了第 4 步的确切 URL 编号列表，各带标题和（若知）日期。

写作纪律：
- 区分**事实**（有引用）和**你的推断**（标"推断："或"可能"）。绝不把推理当成有出处的事实。
- 证据薄就直说——"只找到一个来源，暂作参考"是诚实且合格的结论。
- 不注水。某个子问题没搜到就报告这个空缺，而不是拿废话填。

## 反模式（绝不要做）
- 从训练记忆里作答、事后再贴引用。
- 引用一个你本次并没真抓取过的 URL。
- 把 N 个都引同一篇通稿的博客当成 N 个独立佐证。
- 因为"我很确定"就跳过第 4 步。
- 把一个 Task 子代理当黑盒跑整个研究、让用户干等。
